## 需求
### 需求:系统可以展示首页时间线
系统必须在首页展示帖子时间线，并默认按创建时间倒序排列。

#### 场景:访客浏览首页
- **当** 用户访问首页时间线
- **那么** 系统返回按最新优先排序的帖子列表

### 需求:时间线必须包含帖子展示所需信息
系统必须为每条时间线帖子提供作者基础信息、帖子内容、创建时间、点赞数量，以及当前用户是否已点赞的状态。

#### 场景:已登录用户查看时间线
- **当** 已登录用户请求首页时间线
- **那么** 每条帖子都必须包含该用户对应的点赞状态

#### 场景:未登录用户查看时间线
- **当** 未登录用户请求首页时间线
- **那么** 系统仍必须返回帖子及作者信息，但点赞状态必须表现为未登录可读状态
