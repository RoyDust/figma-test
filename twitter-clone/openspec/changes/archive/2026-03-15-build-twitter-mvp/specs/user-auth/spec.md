## 新增需求

### 需求:用户可以注册账户
系统必须允许访客使用唯一用户名、唯一邮箱和密码创建账户，并为后续登录保存安全的凭证信息。

#### 场景:成功注册
- **当** 访客提交唯一的用户名、邮箱和符合规则的密码
- **那么** 系统创建新用户账户并返回可继续登录的成功结果

#### 场景:用户名或邮箱重复
- **当** 访客提交已存在的用户名或邮箱
- **那么** 系统必须拒绝注册并返回明确的重复提示

### 需求:用户可以登录和退出
系统必须允许已注册用户登录并建立会话，也必须允许用户安全退出。

#### 场景:成功登录
- **当** 已注册用户提交正确的账号凭证
- **那么** 系统建立有效会话并允许访问受保护操作

#### 场景:登录失败
- **当** 用户提交不存在的账户信息或错误密码
- **那么** 系统必须拒绝登录并返回统一的失败提示

### 需求:未登录用户禁止执行受保护操作
系统必须禁止未登录用户执行发帖、点赞等受保护操作。

#### 场景:访客尝试发帖
- **当** 未登录用户提交创建帖子请求
- **那么** 系统必须拒绝该请求

#### 场景:访客尝试点赞
- **当** 未登录用户提交点赞请求
- **那么** 系统必须拒绝该请求
